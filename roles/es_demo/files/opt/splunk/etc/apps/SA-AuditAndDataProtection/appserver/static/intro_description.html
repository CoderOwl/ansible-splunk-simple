Data Protection ensures that log data has not been tampered with or damaged between index time (time it was received) and search time. This allows Splunk to support non-repudiation by providing proof of the integrity of the data. Data Protection can provide proof that the event data has not been corrupted and identify data that has been tampered with or damaged.

Tamper Detection

A number of mechanisms are provided for detecting tampering of event data:
Event Hashing
Applies a hash to indexed data and confirms that the data has not been tampered with at search time. By default, Data Protection hashes all correlated events.
Audit Event Signing
Audit event signing applies a digital signature to indexed data to confirm that the data has not been tampered with and uses a sequence ID to identify missing events. Audit event signing only applies to the Splunk audit log which includes activity within Splunk (such as system startup, adding/removing users, etc.).
IT Data Signing
IT data signing applies a digital signature to indexed data to confirm that the data has not been tampered with. IT data signing is similar to audit event signing except that it applies to external data.

Anonymizing Sensitive Data

In addition to data integrity, Splunk provides a mechanism to anonymize sensitive data within logs such as credit card numbers, PII (personally identifiable data), passwords, etc.
