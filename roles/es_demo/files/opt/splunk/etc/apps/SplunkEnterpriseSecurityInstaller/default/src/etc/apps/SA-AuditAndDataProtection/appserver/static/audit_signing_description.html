Protecting Splunk's Audit Data with Audit Signing

Audit event signing ensures that Splunk's audit log events have not been tampered with and identifies gaps in the sequence of events (caused when an event is deleted).

Configuring Audit Signing

Audit signing is configured in audit.conf. To enable audit signing, specify the public and private key to use. By default, Splunk creates a default private and public key in $SPLUNK_HOME/etc/auth/audit that can be used for audit signing. See splunk>docs for more information.
