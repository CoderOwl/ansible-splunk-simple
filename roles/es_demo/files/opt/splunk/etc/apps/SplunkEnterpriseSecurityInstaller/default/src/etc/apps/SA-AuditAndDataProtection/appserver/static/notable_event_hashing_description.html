Protecting Notable Events with Event Hashing

Correlated events are hashed with a cryptographic algorithm (SHA256) at index time in order to ensure the events displayed at search time have not been changed (i.e. are the same as the events received at index time). Events that match the hash (integrity is verified) will be indicated as valid or tampered in the raw data view per the following:

Valid Tampered! Gap Can't validate!

Event hashing is configured in the file audit.conf. Event hashing can be specified with whitelists (hash events that match given parameters) or blacklists (hash all events except those that match given paramaters).

See splunk>docs for detailed information regarding configuring event hashing.
